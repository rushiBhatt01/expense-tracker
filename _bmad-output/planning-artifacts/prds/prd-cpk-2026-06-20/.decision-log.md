# Decision Log: Smart Expense Splitter

This file logs all major product design decisions, scope changes, and assumptions.

## [2026-06-20] Initial Setup
- **Decision:** Initialized PRD for Smart Expense Splitter.
- **Rationale:** User requested a Splitwise-like expense tracker prototype to settle debts with the minimum number of transactions.

## [2026-06-20] Tech Stack & Project Stakes
- **Decision:** Calibrated as a Hobby project using the MERN stack (MongoDB, Express, React, Node.js).
- **Rationale:** User specified a MERN stack hobby project on the Fast Path.
- **Impact:** The PRD will target a 2-page lightweight scope focusing on direct group sharing, simple APIs, and greedy debt simplification without complex enterprise-level features.

## [2026-06-20] User Authentication, Input UI, and ACID DB Transactions
- **Decision:** Integrate user login via Clerk Service.
- **Rationale:** Secure group spaces, track user-created groups, and associate user accounts with group members.
- **Decision:** Prioritize interactive React Form UI with modal popups for adding participants and expenses.
- **Rationale:** Better UX for real-time mobile and web entries compared to text parsing.
- **Decision:** Ensure ACID database operations using MongoDB transactions.
- **Rationale:** Guarantee data consistency across MongoDB collections (e.g., matching expenses and user net balances) under concurrent updates.

## [2026-06-20] PRD Finalization
- **Decision:** Finalized the PRD draft and closed the authoring phase.
- **Rationale:** All core decisions and requirements (Clerk integration, interactive React form, MongoDB ACID transactions, greedy simplifying algorithm) have been documented, and the validation report graded the document as Excellent.
- **Next Step:** Move to solutioning/architecture or implementation.
